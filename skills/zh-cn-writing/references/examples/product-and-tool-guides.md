# Product and tool guide examples

These normalized examples capture reusable structure and style without reproducing distinctive source prose. Source articles are not factual authorities.

## Example: Postman 使用指南

- Document type: Product and tool guide
- Subject: APIs and developer tools
- Audience: API consumers and integration developers
- Tone: Practical and explanatory
- Formality: Medium to high
- Length: Medium
- Source: `temp/writing-examples/21-0128_Postman.md`

### Structural characteristics

- 从实际问题切入，说明工具定位、核心工作流、示例、限制和适用场景。
- 从 API 调试问题进入安装、核心操作和示例工作流。

### Reusable style characteristics

- 以任务而不是功能清单为主线；避免没有依据的宣传性判断。
- 以任务串联功能，保留产品界面和术语的官方名称。

- Audience: Developers and site operators
- Tone: Explanatory and practical
- Formality: Medium to high
- Length: Medium

### Structural characteristics

- 从实际问题切入，说明工具定位、核心工作流、示例、限制和适用场景。
- 先说明使用场景和能力，再给出配置或接入路径。

- 减少宣传性形容词，以可验证功能和限制为中心。

## Example: Go 辅助脚本使用说明

- Document type: Product and tool guide
- Subject: Go and desktop tooling
- Audience: Go developers
- Tone: Practical and concise
- Formality: Medium to high
- Length: Medium
- Source: `temp/writing-examples/Go/23-1101_Go_Scripts.md`

### Structural characteristics

- 从实际问题切入，说明工具定位、核心工作流、示例、限制和适用场景。
- 按脚本目的、输入、运行方式和结果组织多个工具。

### Reusable style characteristics

- 以任务而不是功能清单为主线；避免没有依据的宣传性判断。
- 每个工具保持独立小节，命令和路径精确呈现。
