# API and integration guide examples

These normalized examples capture reusable structure and style without reproducing distinctive source prose. Source articles are not factual authorities.

## Example: HTML5 全屏 API 使用说明

- Document type: API and integration guide
- Subject: Frontend and web performance
- Audience: Frontend developers
- Tone: Explanatory and practical
- Formality: Medium to high
- Length: Medium
- Source: `temp/writing-examples/Frontend/18-0102_HTML5.md`; `temp/writing-examples/TCloud/18-0102_HTML5_TCloud.md`

### Structural characteristics

- 按使用场景、前置条件、接口或流程、请求响应、错误和完整示例组织。
- 从能力说明进入 API、兼容处理和调用示例。

### Reusable style characteristics

- 接口名和字段保持精确；用稳定术语描述行为与约束。
- 保留浏览器 API 名称，使用代码对照解释行为。

## Example: Web API 项目说明

- Document type: API and integration guide
- Subject: APIs and developer tools
- Audience: API consumers and integration developers
- Tone: Formal and reference-oriented
- Formality: Medium to high
- Length: Medium
- Source: `temp/writing-examples/Pages/21-1103_API.md`

### Structural characteristics

- 按使用场景、前置条件、接口或流程、请求响应、错误和完整示例组织。
- 以项目介绍、接口分组、请求响应和示例组织。

- 接口名和字段保持精确；用稳定术语描述行为与约束。
- 使用稳定的接口模板，便于读者快速定位输入与输出。
